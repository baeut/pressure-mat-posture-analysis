## 一、串口通信协议

### 常规数据包（每100ms）
```json
{"t":12345,"s":[12.3,45.6,...,34.5],
 "p":1,"pn":"端正坐姿","sit_m":30,"bat":3.85}
```

| 字段 | 说明 |
|:-----|:------|
| t | 时间戳 (ms) |
| s | 6路传感器压力值 |
| p | 坐姿编号 0~7 |
| pn | 坐姿名称 |
| sit_m | 连续坐的分钟数 |
| bat | 电池电压（仅电池版） |
| changed | 坐姿是否变化 |
| alert | 久坐警报（仅触发时） |

### 坐姿编号映射
| p | 坐姿 |
|:-:|:-----|
| 0 | 无人 |
| 1 | 端正 |
| 2 | 前倾 |
| 3 | 后仰 |
| 4 | 左倾 |
| 5 | 右倾 |
| 6 | 跷左腿 |
| 7 | 跷右腿 |

### 指令协议

| 指令 | 功能 |
|:-----|:------|
| `c` | 校准归零（首次使用/坐垫移位） |
| `r` | 统计报表 |
| `t` | 实时状态 |
| `s` | 重置统计（新用户/新一天） |
| `action` | 触发 D5 动作引脚 |

## 二、seat_bridge.py 使用指南

### 快速开始
```bash
# 基本启动（自动检测 Arduino）
python seat_bridge.py

# 带屏幕弹窗 + 升降桌
python seat_bridge.py --popup --desk-relay

# 全面启用
python seat_bridge.py --popup --webhook http://localhost:3000/hook --wechat
```

### 完整选项
| 参数 | 说明 |
|:-----|:------|
| `--port COM3` | 指定串口（默认自动检测） |
| `--popup` | Windows 屏幕弹窗 |
| `--exec C:\script.bat` | 久坐时执行脚本 |
| `--webhook http://...` | HTTP POST 通知 |
| `--desk-relay` | 升降桌继电器模式 |
| `--desk-api http://...` | 升降桌 API 模式 |
| `--wechat` | 微信推送 |

---

## 三、升降桌联动

| 方式 | 说明 |
|:-----|:------|
| **D5 继电器** | Arduino D5 → 继电器模块 → 并联升降桌按钮 |
| **API 模式** | `seat_bridge.py` 通过 HTTP API 控制智能升降桌 |

**触发条件（可在 `seat_bridge.py` 配置）：**
- 连续久坐 > 45 分钟 → 触发站立提醒（同时桌升到站立位）
- 坐下 > 5 分钟 → 自动降到坐姿位
- 手动打断：下一次站立提醒重置计时
